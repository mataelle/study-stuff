Данные описывают иcторию посещений пользователями страниц. Формат: текстовый, поля разделены табуляциями:
user_id, timestamp, url, diff_time
user_id – идентификатор пользователя, timestamp – время в секундах, url – адрес страницы, diff_time – время, проведенное на странице, в секундах.
 
Данные необходимо преобразовать с помощью Hadoop (MapReduce), положить в удобном для решения задачи виде в HBase и уметь получать из HBase результат. Таким образом, решение состоит из двух программ: обработчика, который преобразует исходные данные и сохраняет результат, и клиента (вьювера) для отображения результата. При этом клиент может содержать дополнительную логику по преобразованию данных (если необходимо). Обе программы – консольные приложения, вьювер выводит результат на консоль.

Клиент должен уметь выводить распределение количества хитов (просмотров страниц) по доменам за указанный интервал времени. На входе получает интервал времени (дни начала и окончания интервала), на выходе набор пар: домен – доля хитов за данный период на этот домен.
